# Инструкция по деплою и интеграции Telegram Mini App «DA!CAR»

Этот проект представляет собой высокопроизводительное клиентское приложение (SPA) на базе **React 19**, **Vite** и **Tailwind CSS**. Поскольку у проекта нет выделенного серверного бэкенда (все данные и расчеты происходят на клиенте), его деплой предельно прост и экономичен.

---

## 1. Решение ошибки 404 на Vercel

### В чем была проблема?
При деплое SPA-приложений (Single Page Applications) на Vercel часто возникает ошибка **404 Not Found** при обновлении страницы или попытке перейти по прямым ссылкам (например, `/profile`). Vercel пытается найти физический файл на сервере по этому пути, но его нет, так как роутинг является виртуальным (клиентским).

### Как мы это решили?
В корень проекта добавлен файл `vercel.json` со следующим содержимым:
```json
{
  "cleanUrls": true,
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
Этот файл принудительно направляет любые запросы на `index.html`, позволяя приложению самостоятельно управлять состоянием и страницами.

### Настройки в панели управления Vercel:
При импорте репозитория из GitHub убедитесь, что указаны следующие параметры:
- **Framework Preset**: `Vite` (Vercel определит его автоматически).
- **Build Command**: `npm run build` (или `vite build`).
- **Output Directory**: `dist`.

## 1.5. Деплой на Cloudflare Pages (Рекомендуемый способ ⚡)

Cloudflare Pages — это лучший бесплатный, сверхбыстрый хостинг с глобальной CDN для статических сайтов и Telegram Mini Apps.

### Шаг 1. Прямой импорт из GitHub (Автоматический деплой)
1. Авторизуйтесь на сайте [Cloudflare Dashboard](https://dash.cloudflare.com/).
2. Перейдите в левом меню в раздел **Workers & Pages** -> **Pages** -> **Create a project** -> **Connect to Git**.
3. Выберите ваш GitHub-репозиторий с проектом.
4. Настройте параметры сборки (Build settings):
   - **Framework preset**: `Vite` (определится автоматически).
   - **Build command**: `npm run build`
   - **Build output directory**: `dist`
5. Нажмите **Save and Deploy**. Теперь при каждом вашем `git push` сайт будет автоматически собираться и обновляться за 15 секунд!

### Шаг 2. Настройка роутинга (Решение проблемы 404 при обновлении страниц)
В SPA-приложениях роутинг полностью клиентский. Чтобы при обновлении страниц (например, `/profile`) не возникала ошибка 404, в корень собранной папки `dist/` должен копироваться файл `_redirects`.
Мы уже создали файл `/public/_redirects` со следующим содержимым:
```text
/*    /index.html   200
```
При сборке Vite автоматически скопирует его в `dist/`, гарантируя, что роутинг на Cloudflare Pages будет работать безупречно!

## 2. Деплой на обычном VPS/VDS сервере (Ubuntu / Linux)

Так как проект является статическим веб-сайтом, наилучший, самый быстрый и безопасный способ его запуска — раздача собранных файлов через веб-сервер **Nginx** (или **Apache**). Вам даже не нужно постоянно держать запущенным Node.js-процесс.

### Шаг 1. Сборка проекта
Выполните сборку на сервере или локально перед отправкой:
```bash
# Установка зависимостей
npm install

# Сборка в статические файлы
npm run build
```
После этого в корне проекта появится папка `dist/` со всеми готовыми файлами.

### Шаг 2. Настройка Nginx
Установите Nginx (если не установлен):
```bash
sudo apt update
sudo apt install nginx
```

Создайте или отредактируйте конфигурационный файл для вашего домена (например, `/etc/nginx/sites-available/dacar`):
```nginx
server {
    listen 80;
    server_name your-domain.com; # Укажите ваш домен или IP-адрес

    root /var/www/dacar/dist; # Путь к собранной папке dist на сервере
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Логирование и кэширование статики
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|otf)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
```

Активируйте конфигурацию и перезапустите Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/dacar /etc/nginx/sites-enabled/
sudo nginx -t # Проверка синтаксиса
sudo systemctl restart nginx
```

## 3. Деплой на обычном (Shared) хостинге (cPanel, FTP)

Если вы используете простой хостинг (например, Beget, Reg.ru, Timeweb и др.):
1. Запустите команду сборки локально на своем компьютере:
   ```bash
   npm run build
   ```
2. Подключитесь к вашему хостингу через FTP-клиент (например, FileZilla) или откройте файловый менеджер в панели хостинга.
3. Скопируйте **содержимое** локальной папки `dist/` (а не саму папку) в корневую директорию вашего сайта на хостинге (обычно это папка `public_html` или `httpdocs`).
4. Если на хостинге возникнет проблема 404 при обновлении страниц, создайте в корневой директории файл `.htaccess` (для серверов Apache) со следующим кодом:
   ```apache
   <IfModule mod_rewrite.c>
     RewriteEngine On
     RewriteBase /
     RewriteRule ^index\.html$ - [L]
     RewriteCond %{REQUEST_FILENAME} !-f
     RewriteCond %{REQUEST_FILENAME} !-d
     RewriteRule . /index.html [L]
   </IfModule>
   ```

## 4. Настройка Telegram Bot & API Key (для Telegram Mini App)

Для работы Telegram Mini App вам нужно зарегистрировать бота в Telegram и привязать к нему ваше приложение.

### Шаг 1. Создание Бота через `@BotFather`
1. Откройте Telegram и найдите официального бота **`@BotFather`**.
2. Отправьте команду `/newbot` для создания нового бота.
3. Введите название бота (например, `DA!CAR Импорт`) и юзернейм бота, оканчивающийся на `_bot` (например, `dacar_import_bot`).
4. `@BotFather` пришлет вам сообщение с **API токеном** (например, `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`).
   * *Примечание*: Данный токен понадобится, если вы решите добавить серверные Telegram-уведомления или отправку заявок в чат менеджера.

### Шаг 2. Создание Mini App (Web App)
Чтобы пользователи могли запускать приложение прямо внутри Telegram:
1. В диалоге с `@BotFather` отправьте команду `/newapp`.
2. Выберите созданного бота, для которого вы хотите сделать Mini App.
3. Введите название для Mini App (например, `DA!CAR Салон`).
4. Введите краткое описание приложения.
5. Загрузите квадратное изображение/иконку (размер 640x640 пикселей) и GIF-анимацию (по желанию).
6. **Ключевой шаг**: Введите URL вашего приложения. Сюда нужно вставить адрес, который вы получили после деплоя (например, `https://dacar-your-app.vercel.app` или `https://your-domain.com`).
7. Укажите короткое имя для ссылки приложения (например, `app` или `salon`).

В результате вы получите ссылку вида:
`t.me/your_bot/app` или кнопку запуска в меню бота!

## 5. Дополнительно: Отправка заявок в Telegram-чат группы/канала

Так как приложение полностью клиентское, вы можете настроить отправку заявок на покупку авто напрямую из браузера пользователя в ваш Telegram-чат с помощью стандартного Telegram Bot API.

Для этого в коде отправки заявки (например, при подтверждении заказа) можно использовать простой HTTP-запрос к API Telegram:

```typescript
const BOT_TOKEN = 'ВАШ_API_ТОКЕН_ОТ_BOTFATHER';
const CHAT_ID = 'ID_ВАШЕГО_ЧАТА_ИЛИ_КАНАЛА'; // Можно получить у @userinfobot или узнать ID группы

const sendTelegramNotification = async (message: string) => {
  try {
    await fetch(`https://api.telegram.org/bot${BOT_TOKEN}/sendMessage`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        chat_id: CHAT_ID,
        text: message,
        parse_mode: 'HTML',
      }),
    });
  } catch (error) {
    console.error('Ошибка отправки уведомления в Telegram:', error);
  }
};
```
